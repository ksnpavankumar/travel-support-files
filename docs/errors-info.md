  You are getting this error because Jackson is trying to serialize an internal object (HttpURLConnection) that comes from the Amadeus SDK response.

The important thing to understand:

👉 You must never return the Amadeus SDK response object directly from your controller.

Jackson tries to inspect every field inside it and eventually reaches HttpURLConnection, which has two setters:

setFixedLengthStreamingMode(int)
setFixedLengthStreamingMode(long)

Jackson cannot decide which one to use, so it throws:

IllegalArgumentException: Conflicting setter definitions
What is happening in your code

You likely have something like this:

@GetMapping("/availability")
public FlightAvailabilities getAvailability() throws ResponseException {
    return amadeus.shopping.availability.flightAvailabilities.get(params);
}

Spring Boot tries to convert FlightAvailabilities → JSON using Jackson.

But the object contains network internals, including HttpURLConnection.

Jackson fails.

Correct way to fix it

Use the body of the response, not the SDK object.

Fix
@GetMapping("/availability")
public String getAvailability() throws ResponseException {

    FlightAvailabilities response =
        amadeus.shopping.availability.flightAvailabilities.get(params);

    return response.getBody();
}

getBody() already contains the JSON returned by Amadeus.

Spring can safely return it.

Even better (recommended)

Convert it into a DTO.

@GetMapping("/availability")
public Object getAvailability() throws Exception {

    FlightAvailabilities response =
        amadeus.shopping.availability.flightAvailabilities.get(params);

    ObjectMapper mapper = new ObjectMapper();

    return mapper.readValue(response.getBody(), Object.class);
}

or map to your own class.
